# 16 — Boas práticas

## Objetivo

Síntese operacional por canal, com links para as regras detalhadas.

## Digital (site / app)

1. Usar tokens `oboya-*` ([10-design-tokens.md](./10-design-tokens.md)).
2. Logo via componente oficial; clear space ([03-logo.md](./03-logo.md)).
3. Primaries Blue + Green; accents com disciplina ([04-colors.md](./04-colors.md)).
4. Overlay em heroes fotográficos ([07-imagery.md](./07-imagery.md), [13-accessibility.md](./13-accessibility.md)).
5. Tipografia Display + Body ([05-typography.md](./05-typography.md)).
6. Revisar [brand-audit.md](./brand-audit.md) antes de shipar UI nova.

## Peças gráficas / print

1. Escala 64/72, 48/56, 32/40 ±8px ([08-layout.md](./08-layout.md)).
2. Um accent ≤ 25%.
3. Qualidade de impressão em fotos de produto.
4. Soft White / White como neutros.

## Social

1. Exportar com safe zones corretas ([08-layout.md](./08-layout.md)).
2. Não usar Dark Blue como cor de conteúdo institucional em social ([04-colors.md](./04-colors.md)).
3. Texto legível sobre imagem (overlay).
4. Frescor visual de hortifrúti ([07-imagery.md](./07-imagery.md)).

## Feiras

1. 80 cm livres na base do booth.
2. QR visível; imagens print-ready.
3. Layout adaptável ao espaço real.

## Email

1. Seguir hierarquia do exemplo de assinatura.
2. Logo + dados de contato claros.
3. Cores estruturais; evitar accents excessivos em assinatura.

## Boas práticas gerais

- Restrições explícitas > preferência estética.
- Marcar o que é oficial vs. recomendação em briefs.
- Atualizar [20-changelog.md](./20-changelog.md) quando a documentação mudar.
