# 19 — Checklist de entrega

## Objetivo

Listas práticas por tipo de entrega para validar conformidade com o Brandbook 2026.

## Toda peça (mínimo)

- [ ] Main Blue + Main Green presentes como estrutura
- [ ] Neutros = White e/ou Soft White
- [ ] No máximo um accent (Light Yellow **ou** Orange Red), ≤ 25%
- [ ] Sem Light Yellow + Orange Red juntos
- [ ] Sem texto amarelo claro sobre branco
- [ ] Dark Blue não usado como fundo large-format indevido
- [ ] Logo com clear space (= altura do “O”)
- [ ] Tipografia: Display nos títulos, Plus Jakarta no corpo (peças gráficas)

## UI / web

- [ ] Tokens `oboya-*` / semânticos documentados
- [ ] Sem hex fora da paleta em superfícies de marca
- [ ] Logo via `Logo` component; variante light em fundo escuro
- [ ] Overlay se texto sobre foto
- [ ] Contraste AA nos pares críticos
- [ ] Review contra [brand-audit.md](./brand-audit.md) se tocar tokens/cores

## Social

- [ ] Formato correto (1920 / 1728 / 1440 / 1350 / 1080)
- [ ] Conteúdo crítico dentro da safe zone
- [ ] Sem Dark Blue institucional como protagonista
- [ ] Produtos frescos / embalagens impecáveis

## Print / merchandising

- [ ] Escala 64/72, 48/56, 32/40 (±8px se justificado)
- [ ] Imagens em qualidade de impressão
- [ ] Cores em CMYK conforme Brandbook quando for offset

## Feira / booth

- [ ] 80 cm livres na base
- [ ] QR fácil de ver
- [ ] Imagens print-ready
- [ ] Layout adaptado ao espaço real

## Email / assinatura

- [ ] Logo legível
- [ ] Nome, cargo, contatos hierarquizados
- [ ] Cores estruturais; accents mínimos

## Documentação

- [ ] Mudanças de regra refletidas em `docs/brand/`
- [ ] [20-changelog.md](./20-changelog.md) atualizado
- [ ] Lacunas marcadas como “não informado” (não inventadas)
